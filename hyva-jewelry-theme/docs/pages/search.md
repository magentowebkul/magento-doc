# Search Results Page

The **Search Results Page** shows products that match the shopper's search word. It uses the same clean product cards as category pages.

Example: when a shopper searches `ring`, the page can show matching ring products.

## What shoppers see

| Area | What it does |
| --- | --- |
| **Search title** | Shows the search term, such as `Search results for: 'ring'`. |
| **Result count** | Shows how many products were found. |
| **Product grid** | Shows matching products with image, name, price, stock, and badges. |
| **Filters** | Lets shoppers narrow results by jewellery attributes. |
| **Pagination** | Lets shoppers move through many results. |
| **Show per page** | Lets shoppers show 12, 24, or 36 products per page. |
| **Product actions** | Shows Wishlist, Compare, Quick View, and Add to Cart where available. |

## Search filters

Search results can use filters like:

- Jewelry Material
- Metal Color
- Stone
- Diamond Shape
- Occasion
- Gender
- Purity
- Ring Size
- Finish
- Certification

These filters help shoppers find the right product after searching.

## Header search

The header has a search field for desktop and mobile. Shoppers can type a product word, such as `ring`, `gold`, `pearl`, or `bracelet`.

Good search results need good product data. Add clear product names, descriptions, and attribute values.

## How to check search

1. Click the search icon or search field in the header.
2. Search for a common product word, such as `ring`.
3. Check that products show on the results page.
4. Test filters on the search page.
5. Open a product from the results.
6. Test Wishlist, Compare, Quick View, and Add to Cart.
7. Search for a word with no results and check the message.
8. Check search on mobile.

## Good practice

- Use product names that shoppers understand.
- Add important words to product descriptions.
- Fill jewellery attributes like material, stone, size, and occasion.
- Keep product status, stock, and visibility correct in Magento Admin.
- Test search after adding new products.
